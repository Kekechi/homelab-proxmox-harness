# /handoff

Package a production plan for operator handoff. Claude cannot apply production plans — the operator must review and apply manually.

## What This Command Does

1. Verifies `terraform/production.tfplan` exists
2. Runs `terraform show production.tfplan` to capture the human-readable diff
3. Assembles a structured handoff document with full context

## Usage

```
/handoff
```

Run this after `make plan ENV=production` has generated `production.tfplan`.

## Handoff Document Format

Produce a markdown document with these sections:

### 1. Change Summary

One-paragraph description of what this change does and why. Reference the original request or plan context from the conversation.

### 2. Resources Affected

| Action | Resource | Key Details |
|--------|----------|-------------|
| create/modify/destroy | `module.name.resource.id` | Size, IP, pool, etc. |

### 3. Plan Diff

Include the full output of `terraform show production.tfplan` in a code block.

### 4. Risk Assessment

- **Destructive actions:** List any destroy or replace operations
- **Rollback plan:** How to undo if something goes wrong
- **Downtime:** Whether existing resources will be interrupted
- **Dependencies:** Other systems affected (Ansible inventory, DNS, monitoring)

### 5. Apply Instructions

Exact commands for the operator to run:

```bash
# From the repo root, inside the dev container (or on the operator's machine)
cd terraform

# Verify the plan matches expectations
terraform show production.tfplan

# Apply
terraform apply production.tfplan

# Verify
terraform state list
terraform output -json
```

### 6. Post-Apply Steps

- Configuration updates needed (inventory, DNS, monitoring)
- Ansible playbooks to run
- Verification checks

## After Handoff

The handoff document should be presented to the user for them to deliver to the operator via their preferred channel (Slack, email, PR comment, etc.). Claude does not send the handoff directly.

## Constraints

- This command is read-only — it does not modify any files
- Production plan files are generated by `make plan ENV=production`
- If `production.tfplan` does not exist, instruct the user to run `make plan ENV=production` first
